# 审计智能体贯穿案例·Toy Data Pack

## 这是什么

这是一组专为课堂设计的虚构数据。它把“42,000笔报销”缩小为26笔可以手工理解的样本，但保留了真实审计任务中的关联方式和证据分散问题。

## 目录

- `data/`：9个本期待审CSV数据文件
- `classroom_training/ml_training_examples.csv`：300条独立历史标注案例（240条训练、60条验证）
- `policy_documents/`：2份虚构制度文档
- `instructor_answer_key/expected_findings.csv`：讲师答案，列出A—F六个贯穿情形
- `toy_audit_case.xlsx`：同一组数据的Excel工作簿版本
- `data_dictionary.md`：文件用途、记录数和关联键

## 六个贯穿情形

| 情形 | 报销号 | 教学目的 |
|---|---|---|
| A 重复发票 | BX-41610 / BX-41902 | 规则擅长确定性匹配 |
| B 合理例外 | BX-41002 | 单看超标会误报，需要制度与审批语境 |
| C 拆分报销 | BX-41881—BX-41884 | 单笔均正常，组合模式可疑 |
| D 票据修改 | BX-42306 | 问题藏在图像和二维码金额矛盾中 |
| E 个人消费伪装 | BX-42519 | 需要综合票据语义、CRM和日历 |
| F 行程矛盾 | BX-42017 | 需要主动跨系统取证并形成闭环 |

## 建议的课堂使用顺序

1. 先只发放`expense_claims.csv`，让学员做人工筛查。
2. 讲机器学习时使用`ml_training_examples.csv`解释训练、验证、拟合和阈值；不要使用26笔本期待审数据训练模型。
3. 再逐步发放发票、审批、行程、OCR等数据，体验证据如何改变判断。
4. 最后打开讲师答案，对照不同技术阶段能发现什么、遗漏什么。

## 两类数据必须严格区分

- **历史训练数据**：300条虚构案例，包含审计人员已经确认的标签，只用于课堂训练和验证。
- **本期待审数据**：26笔报销及其关联证据，用于训练完成后的推理、调查和人工复核。

把本期待审事项的答案放入训练集会造成数据泄漏，得到的漂亮结果不能代表模型能够处理新事项。

> 所有员工、客户、商户、金额、票据和制度均为教学用虚构内容，不含真实个人或企业数据。
