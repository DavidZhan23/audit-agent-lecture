## 趣味支线讲法（5—7分钟）

1. **约1分钟｜先讲动机。** 指着照片介绍“课堂虚构故事”：笑雨生活在热闹却步履匆匆的都市，每天与许多人擦肩，却因为总被邻居叫错名字而觉得自己像“一张没有被保存的面孔”；向小伙伴骐源诉苦后，发现骐源也有同样烦恼。两人决定做一个只认识他们俩、并把其他人拒之圈外的模型。模型一次次显示“笑雨”时，用“能被你认识，真好啊”落下情绪，再以“世界很大，而我想被认出来”转入模型机制。
2. **约1—2分钟｜走训练流程。** 强调一张照片不足以代表一个人；笑雨与骐源都需要角度、光线、表情不同的照片，更多邻居和陌生人则组成“其他”。把前面的数字ANN原样迁移：像素输入、多层表示、Softmax、Loss、反向传播；变化的只是输入照片和输出标签。
3. **约1分钟｜讲精妙点。** 模型的世界被刻意缩成笑雨 / 骐源 / 其他三类：既要分清两位熟人，也要学会拒绝圈外人物；低于0.70则不硬猜。
4. **约1—2分钟｜模型亮相。** 点击“让模型认认笑雨”运行内置照片；若时间允许，再请现场学员换一张照片对比三类概率。
5. **约30秒｜反转收束。** 模型能把像素分进类别，却不知道“被认错为什么让人难过”；识别内容不等于理解语言、制度或业务，由此过渡到LLM。
