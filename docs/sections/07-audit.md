# 07 · Agent在审计中的应用

- **部分：** `#part-3` · 第三部分 · 核心
- **锚点：** `#audit`
- **时间：** 25分钟
- **定位：** 用审计交付、证据、权限与职业判断要求约束通用Agent

## 教学目标

- 先定义审计交付，再选择模型和工具
- 判断一个任务适合Agent、只需规则、不应自动化或目标过宽
- 把规则、ML、ANN、LLM和Agent组合成端到端流程
- 保证事实、标准和模型解释都可回到原始来源
- 区分可自动筛查、受控调查与必须人工完成的职业判断
- 设计离线、影子、试点和扩围四阶段评价路径

## 八步正文

1. **定义审计交付：** 五字段可复核疑点证据包——事实、适用标准、原始证据、不确定性、核查建议。
2. **选择窄场景：** 以任务边界、数据可得、证据可定义、重复频率和人工复核五维评分；差旅初审适合作为试点，重复发票优先规则，自动签发意见被禁止，“万能Agent”应拆分。
3. **组合技术：** 确定性用规则，统计排序用ML，图像感知用ANN，语言语境用LLM，跨系统动态调查用Agent。
4. **连接数据与制度：** 九张表、两份制度，通过claim/invoice/trip/employee键关联；保留权限、版本、时间和失败状态。
5. **端到端流程：** 任务受理 → 规则筛查 → 模型增强 → Agent调查 → 证据包 → 人工复核；提供一段可运行Python。
6. **形成证据包：** 以BX-42017逐项检查事实、标准、来源、不确定性和建议。
7. **治理控制：** 提示注入、隐私保密、越权工具、事实幻觉、职责混淆、版本与漂移。
8. **评价上线：** 离线基准 → 影子运行 → 小范围试点 → 受控扩围；每阶段同时设置质量与风险门槛。

## 自动化边界

| 可以自动或受控执行 | 必须由审计人员承担 |
|---|---|
| 字段完整性、重复和阈值检查 | 评价证据是否充分适当 |
| 历史模式排序、图像与语言提取 | 访谈、沟通和解释冲突 |
| 白名单内只读查询与证据整理 | 重大定性、底稿批准和审计意见 |

## 必须留下的一句话

> 审计智能体的直接产出是可复核疑点，不是自动审计结论；先证明做得对且查得回，再证明做得快。
