# 06 · Agent基础与架构

- **部分：** `#part-2` · 第二部分 · 核心
- **锚点：** `#agent`
- **时间：** 35分钟
- **定位：** 全课通用Agent核心章；不依赖审计专业知识也能成立

## 教学目标

- 区分普通程序、工作流、LLM应用和Agent
- 理解Agent不是更大的模型，而是完整运行系统
- 说出六块架构及其输入、产出和失败风险
- 理解工具契约、状态、记忆、轨迹、反馈分支和停止条件
- 能按动作风险选择自主度与人工关口
- 用结果、过程、成本和风险评价Agent，而不是只看回答流畅度

## 七步正文

1. **行动缺口：** 模型说“查航班”不等于系统已经查询。
2. **定义与区分：** Agent围绕目标选择行动、调用工具、读取反馈、更新状态并受控停止。
3. **六块架构：** 目标与策略、LLM决策核、工具层、状态与记忆、编排循环、控制与可观测。
4. **工具契约：** 展示成功、超时、无权限三种结构化返回及不同后续动作。
5. **反馈与停止：** 区分工作状态、长期记忆和运行轨迹；展示成功、预算、失败、风险、人工五类停止。
6. **自主度与控制：** 建议模式、只读调查、受控执行；动作分别划为可自动、需批准、始终禁止。
7. **价值与评价：** 完成率、遗漏、工具成功、事实正确、成本、接管率、安全；同时讲目标漂移、工具误用、状态污染、静默失败和无限循环。

## 贯穿演示

BX-42017先查询航班：

- 行程一致：缩短调查并停止。
- 抵达城市矛盾：扩展查询酒店、CRM和日历。
- 工具失败：记录失败，不将“查不到”解释为“无异常”，按策略重试或转人工。

页面同时提供 `AgentBranchLab` 和可运行Python；代码中的 `choose_next_action(state)` 证明行动来自当前状态，而非固定顺序列表。

## 必须留下的一句话

> Agent不是更大的LLM，而是把模型、工具、状态、循环和控制组织起来的运行系统。
