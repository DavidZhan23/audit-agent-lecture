# 整堂课结构总览

**最后核对代码：** `app/page.tsx` → `export default function Home`
**课程标题：** LLM，Agent基础、架构以及其在审计中的应用
**顶层架构：** 第一部分「大模型和智能体的技术基础」→ 第二部分「Agent基础与架构」→ 第三部分「Agent在审计中的应用」。案例递进只属于第一部分的教学方法，不替代三部分总体架构。

## 课程目标

1. 第一部分从问题出发，讲清规则 → ML → ANN → LLM 为什么逐层出现
2. 第二部分把 Agent 作为独立的系统概念，讲清定义、与LLM的区别、组成模块、工具反馈循环、行业规范、价值和自主边界
3. 第三部分将前两部分组合成审计智能体；当前 `#audit` 为占位，后续逐节设计

## 三部分总体架构

### 第一部分：大模型和智能体的技术基础（`#problem`—`#llm`）

先回答“这些技术为什么会逐层出现”：从问题出发，依次讲普通规则、特征拟合、人工神经网络以及从ANN到LLM。每种新技术都必须由上一种方法解决不了的具体问题引出。

### 第二部分：Agent基础与架构（`#agent`）

全课核心和最长章节，集中回答：Agent是什么、LLM和Agent的区别、Agent模块、工具与反馈循环、行业规范、使用价值以及自主运行边界。BX-42017只是本章用于验证反馈循环的演示，不是本章的顶层结构。

### 第三部分：Agent在审计中的应用（`#audit`）

将前面的通用技术组合成审计智能体，计划讨论场景选择、数据制度工具连接、可复核证据、自动化边界、人工判断以及试点评价。当前保持占位，等待后续逐节设计。

## 能力阶段（`stages`）

| key | 名称 | 核心问题 | 擅长 | 边界 |
|-----|------|----------|------|------|
| `code` | 通俗逻辑与规则 | 条件是否满足？ | 明确逻辑批量执行 | 写不尽的情况 |
| `ml` | 特征拟合（ML） | 像不像历史异常？ | 人工特征上拟合 | 特征靠人；无像素 |
| `nn` | 人工神经网络 | 高维输入里有什么？ | 超多特征表示学习 | 看见≠理解制度 |
| `llm` | 大语言模型 | 合起来意味着什么？ | 语言理解与生成 | 幻觉；不主动取数 |
| `agent` | Agent + LLM | 下一步调用什么、如何停？ | 工具闭环与受控停止 | 须权限与人审 |

## 教学节奏

```text
hero（三部分总体架构）
  → 第一部分：技术基础
      → problem（总体结构 + 第一部分的问题递进方法）
      → code（完全重复：确定性规则）
      → ml（拆分报销：弱信号组合）
      → nn（票据修改：像素；DigitsImageLab + 人脸趣味支线）
      → llm（招待目的：语言与业务语境）
  → 第二部分：Agent基础与架构（40′核心长章）
      → 定义 / 区别 / 模块 / 工具反馈循环 / 规范 / 价值 / 边界 / BX-42017演示
  → 第三部分：Agent在审计中的应用
      → audit（占位 + 简短复盘/自测）
```

## 第一部分内部的案例阶梯

| 层级 | 事项 | 一句话问题 | 新增困难 |
|------|------|------------|----------|
| 规则 | BX-41610 / BX-41902 | 同一张发票被报销两次 | 无；条件可完整表达 |
| ML | BX-41881—84 | 四笔均低于2,000元但集中发生 | 多个弱信号组合 |
| ANN | BX-42306 | 票面286元、平台86元 | 异常藏在高维像素中 |
| LLM | BX-42519 | 客户招待却有儿童餐、生日蛋糕及无拜访记录 | 语言与业务语境 |

BX-42017放在第二部分末尾，用于演示Agent依据工具反馈决定下一步。它不属于第一部分的模型技术阶梯。
